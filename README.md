# Young Developer Roadmap

> Build first. Learn when you need it. Share what you create.

A project-based roadmap designed for young developers (roughly ages 12–18) who love building things.

This is **not** another programming curriculum.

It is a long-term journey that encourages kids to become creators, engineers, and eventually open-source contributors.

---

## Philosophy

Most programming courses teach syntax.

This roadmap teaches **building**.

Instead of learning JavaScript, Python, or C++ chapter by chapter, learners complete meaningful projects and discover new concepts as they need them.

Learning follows building—not the other way around.

---

## The Journey
Builder
↓

Game Developer
↓

Software Developer
↓

Computer Scientist
↓

Open Source Contributor
↓

AI Builder
↓

Independent Engineer

Each stage focuses on creating increasingly ambitious projects while developing engineering habits and problem-solving skills.

---

## Roadmap

| Age | Stage | Goal |
|------|-------|------|
| 12 | Builder | Build games and learn programming fundamentals |
| 13 | Software Developer | Build real applications and APIs |
| 14 | Computer Scientist | Learn algorithms and computer science |
| 15 | Open Source Contributor | Collaborate with developers worldwide |
| 16 | AI Builder | Build AI-powered applications |
| 17–18 | Independent Engineer | Create products with real users |

---

## Core Principles

- Build more than you watch.
- Learn concepts when projects require them.
- Publish everything.
- Document your work.
- Share your ideas.
- Help others.
- Stay curious.
